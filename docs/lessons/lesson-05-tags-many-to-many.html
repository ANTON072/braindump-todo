Day 11: タグ upsert ヘルパー

1. 保存ロジックの設計を言葉で押さえる

「タグ名の配列を受け取り、todo に紐付ける」処理は 3 ステップに分解できます：

  1. INSERT ... ON CONFLICT DO NOTHING — 全タグ名を一括 INSERT。既存の (userId, name) は Lesson 2 で張ったユニーク制約に衝突して静かにスキップされる。
  2. SELECT で引き直す — 新規・既存を問わず、対象タグの id を名前でまとめて取得する。
  3. 中間テーブルへ INSERT — (todoId, tagId) を一括 INSERT。こちらも複合主キーとの衝突は DO NOTHING で流す。

学びどころ：なぜ「INSERT してから SELECT し直す」のか

onConflictDoNothing().returning() は実際に挿入された行しか返しません。既存タグの id は returning では手に入らない。 だから「新規も既存も区別せず、最後に名前で SELECT し直す」のがこのパターンの定石です。 1 件ずつ「SELECT → なければ INSERT」をループする素朴な実装と比べ、クエリ数が件数に依存しない（常に 3 回）のも利点。

2. 実装

src/features/tags/upsert-tags.ts。トランザクションの中から呼べるよう、第 1 引数で tx を受け取る形にします。

import { and, eq, inArray } from "drizzle-orm";
import { db } from "@/db";
import { tags, todosToTags } from "@/db/schema";

type Tx = Parameters<Parameters<typeof db.transaction>[0]>[0];

export function normalizeTagNames(raw: string): string[] {
  const names = raw
    .split(",")
    .map((name) => name.trim().toLowerCase())
    .filter((name) => name.length > 0);
  return [...new Set(names)];
}

export async function attachTagsToTodo(
  tx: Tx,
  userId: string,
  todoId: string,
  tagNames: string[],
) {
  if (tagNames.length === 0) return;

  // 1. あれば衝突スキップ、なければ作成
  await tx
    .insert(tags)
    .values(tagNames.map((name) => ({ userId, name })))
    .onConflictDoNothing();

  // 2. 新規・既存を問わず id を引き直す
  const tagRows = await tx
    .select({ id: tags.id })
    .from(tags)
    .where(and(eq(tags.userId, userId), inArray(tags.name, tagNames)));

  // 3. 中間テーブルに紐付け
  await tx
    .insert(todosToTags)
    .values(tagRows.map((tag) => ({ todoId, tagId: tag.id })))
    .onConflictDoNothing();
}

Day 12: 作成フローへの組み込み

3. スキーマとフォームに tags を足す

todoSchema に 1 フィールド追加（カンマ区切り文字列のまま受ける）：

tagsInput: z.string().max(500).optional(),

フォームには <Input {...getInputProps(fields.tagsInput, { type: "text" })} placeholder="タグ（カンマ区切り: 買い物, 仕事）" /> を追加。チップ型ピッカーは作りません（優先度 4）。

4. createTodo をトランザクション化する

import { attachTagsToTodo, normalizeTagNames } from "@/features/tags/upsert-tags";

export async function createTodo(prevState: unknown, formData: FormData) {
  const userId = await requireUserId();

  const submission = parseWithZod(formData, { schema: todoSchema });
  if (submission.status !== "success") return submission.reply();

  const { tagsInput, ...todoValues } = submission.value;
  const tagNames = normalizeTagNames(tagsInput ?? "");

  await db.transaction(async (tx) => {
    const [todo] = await tx.insert(todos)
      .values({ ...todoValues, userId })
      .returning();
    await attachTagsToTodo(tx, userId, todo.id, tagNames);
  });

  revalidatePath("/todos");
  return submission.reply({ resetForm: true });
}

学びどころ：トランザクションの境界の決め方

「todo は作られたのにタグの紐付けだけ失敗した」という中途半端な状態を許さないため、 todo 本体の INSERT とタグ処理を 1 つの db.transaction に入れます。 ヘルパーが db ではなく tx を受け取る設計にしたのはこのためで、「トランザクションを開く責任は呼び出し側、 中で何をするかは部品側」という分担は実務でもよく使う形です。

Day 13: タグ付きで一覧を取得する

5. リレーショナルクエリ

src/app/todos/page.tsx の取得部を差し替え：

const todoRows = await db.query.todos.findMany({
  where: eq(todos.userId, userId),
  orderBy: [desc(todos.createdAt)],
  with: {
    todosToTags: {
      with: { tag: true },
    },
  },
});

// 中間テーブル 2 段ネストを UI 用にフラット化する
const todosWithTags = todoRows.map((todo) => ({
  ...todo,
  tags: todo.todosToTags.map((link) => link.tag),
}));

表示は shadcn の Badge でタグ名を並べるだけで十分です。

学びどころ：with の 2 段ネストは Lesson 2 の relations の写し絵

with: { todosToTags: { with: { tag: true } } } という形は、relations() で宣言した 「todos → todosToTags → tag」の経路をそのままなぞっています。SQL 的には JOIN 相当の処理を Drizzle が組み立て、結果を型付きのネスト構造で返してくれる——これがリレーショナルクエリ API の価値です。 返り値の型にカーソルを乗せて、todosToTags: { tag: { name: string ... } }[] まで推論されていることを確認してください。

6. （任意）編集時のタグ更新

余力があれば、todo 編集でタグを差し替える処理も書いてみてください。定石は「tx.delete(todosToTags).where(eq(todosToTags.todoId, todoId)) で紐付けを全消し → attachTagsToTodo で再構築」。差分更新より単純で、中間テーブルならではの割り切りです。

チェックポイント

  • 「買い物, 仕事」で todo を 2 件作成 → tags テーブルの行は 2 行のまま増えない（再利用の確認）
  • 一覧の各 todo にタグの Badge が表示される
  • todo を削除 → todos_to_tags の紐付け行が消える（cascade の確認）が、tags 本体は残る
  • 別ユーザーで同名タグ「買い物」を作れる（ユニーク制約が userId 込みであることの確認）

つまずきポイント

  • inArray(tags.name, []) のように空配列を渡すとエラーになる。ヘルパー冒頭の length === 0 ガードを消さないこと。
  • タグ名の正規化（trim・小文字化・重複除去）を怠ると「買い物」と「買い物 」が別タグになる。正規化は保存直前の 1 箇所（normalizeTagNames）に集約する。
  • db.query が undefined になる場合は、Lesson 2 の drizzle(pool, { schema }) で schema を渡し忘れている。
