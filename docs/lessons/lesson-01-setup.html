Lesson 1: プロジェクトセットアップ

Next.js (App Router) / TypeScript / Tailwind / shadcn/ui ｜ 目安 Day 1–2（約 2 時間）

ゴール

pnpm dev で開発サーバーが立ち上がり、shadcn/ui の Button と Card が表示されるトップページがある状態。

Day 1: Next.js の雛形を作る

1. create-next-app

リポジトリ直下（braindump-todo/）で実行します。既存ディレクトリに展開するため . を指定します。

pnpm dlx create-next-app@latest . --typescript --tailwind --app --src-dir --import-alias "@/*"

Turbopack は Yes。リンター／フォーマッタは、本プロジェクトでは ESLint ではなく Biome を採用します（pnpm check で biome check が走る）。create-next-app で Biome を選べればそれを、選べなければリンターなしで進め、後から Biome を導入して構いません。

学びどころ：App Router の頭の切り替え

  • src/app/ 配下のフォルダ構造がそのまま URL になる。src/app/todos/page.tsx → /todos。
  • コンポーネントはデフォルトで Server Component。サーバーで実行されるので、その中で直接 DB を読める（後のレッスンで多用する）。
  • useState やイベントハンドラが必要なときだけ、ファイル先頭に "use client" を書いて Client Component にする。「まず Server、必要なときだけ Client」が王道。

2. 動作確認

pnpm dev

http://localhost:3000 でデフォルトページが表示されれば OK。

Day 2: shadcn/ui を導入する

3. 初期化

pnpm dlx shadcn@latest init

スタイルやベースカラーは好みで（迷ったらデフォルト）。components.json が生成されます。

学びどころ：shadcn/ui は「ライブラリ」ではない

shadcn/ui は npm パッケージとして依存に入るのではなく、コンポーネントのソースコードを自分のリポジトリにコピーする方式です。 pnpm dlx shadcn add button すると src/components/ui/button.tsx が生成され、以後は自分のコードとして自由に編集できます。 「アップデートで壊れない代わりに、アップデートも自分でやる」というトレードオフを選んだツールだと理解しておくと、他プロジェクトでの採用判断にも役立ちます。

4. 今後使うコンポーネントをまとめて追加

pnpm dlx shadcn@latest add button card input label textarea select badge checkbox

5. トップページを差し替える

src/app/page.tsx を最小構成に書き換えて、shadcn のコンポーネントが効いていることを確認します。

import { Button } from "@/components/ui/button";
import { Card, CardContent, CardHeader, CardTitle } from "@/components/ui/card";

export default function Home() {
  return (
    <main className="mx-auto max-w-2xl p-8">
      <Card>
        <CardHeader>
          <CardTitle>braindump-todo</CardTitle>
        </CardHeader>
        <CardContent className="flex gap-2">
          <Button>ログイン</Button>
          <Button variant="outline">新規登録</Button>
        </CardContent>
      </Card>
    </main>
  );
}

学びどころ：Tailwind の使い方はこの 3 パターンで 8 割

  • レイアウト：flex gap-2、grid grid-cols-2、mx-auto max-w-2xl
  • 余白：p-8（padding）、mt-4（margin-top）、数字は 0.25rem 刻み
  • 状態：hover:bg-gray-100、disabled:opacity-50

今回のプロジェクトでは凝ったデザインはしない方針なので、細かいユーティリティは shadcn コンポーネントに任せ、自分で書くのはレイアウトと余白程度に留めます。

6. コミット

ここで一度コミットしておきます（以後、各レッスン末尾でコミットする習慣にします）。

チェックポイント

  • pnpm dev でエラーなく起動する
  • トップページに shadcn の Button（スタイル付き）が表示される
  • src/components/ui/ にコンポーネントのソースが生成されている
  • pnpm exec tsc --noEmit が通る

つまずきポイント

  • create-next-app は空でないディレクトリを嫌がることがある。requirements.md 等がある状態で失敗する場合は、一時的に退避してから実行し、戻す。
  • shadcn の init で Tailwind の設定が見つからないと言われたら、Tailwind v4（CSS ファーストの設定）と v3（tailwind.config.ts）の違いが原因のことが多い。shadcn 公式の Next.js ガイドの手順に従うこと。
