braindump-todo ハンズオントレーニング

対象：requirements.md に定義された 9 ツールのスタック習得＋テスト（Vitest / Playwright）＋運用（CI/CD・可観測性・性能）／ 時間予算：約 1 時間 × 49 日 ＝ 実働 約 49 時間（全 14 レッスン）

このトレーニングは「Todo アプリ＋ブレインダンプ抽出＋タグ（多対多）」を題材に、 Next.js / TypeScript / Tailwind & shadcn/ui / Zod / Drizzle ORM / Conform / Better Auth / OpenAI API / OpenNext + SST を王道・イディオム的な使い方で一巡することを目的とします。凝った実装よりも、後で読み返して学べるコードを優先します。

順序の意図：スコープ優先度（①認証＋CRUD → ②タグ多対多 → ③ブレインダンプ → ④UI作り込み）に沿って、 必達コアが完成した Lesson 6 の直後にデプロイ（Lesson 7）を挟んでいます。 万一 Lesson 8 以降で時間切れになっても「認証つき・タグつき Todo アプリが AWS で動いている」状態が確保されます。 初回デプロイと Better Auth × Drizzle 接続は既知の山場なので、それぞれ 3 時間を確保しています。 またテスト基盤（Lesson 4）をコア実装（Lesson 5・6）の手前に置き、認可・upsert を書いた直後に同じレッスン内で統合テストを書けるようにしています。E2E（Lesson 10）は全機能が揃ってから代表フロー 1 本だけを最後に足します。 最後の Lesson 11〜14 で運用（CI/CD・失敗時の UX・可観測性・性能）を積み、「作れる」から「出して・気づいて・直して・支えられる」へ広げます。ここまでで初めてフルスタックの輪が閉じます。

テスト戦略（ダイヤモンド型）

AI がバックエンドを大量に生成する時代に決定的に効くのは「生成されたコードが正しいと検証できる能力」です。 本教材はダイヤモンド型——中央（実 DB に対する統合テスト）を最も太く、ユニットと E2E を薄く——で組み立てます。

E2E（Playwright）
統合テスト（サービス関数 × 実 DB）
ユニット（純粋関数）

中央（統合）を最も太く。上下は薄く。

  • Server Component は E2E で見る（Next.js 公式が async Server Component はユニットより E2E を推奨）。app/todos/page.tsx は「認証→クエリ→描画」の薄いガワなので、その描画経路は Lesson 10 の実ブラウザで担保する。
  • ロジックはサービス層に抽出し、実 DB で統合テストする。next/* をモックせずに済むよう、認可・作成ロジックを service.ts へ切り出す（Lesson 5・6）。
  • 純粋関数だけユニット（normalizeTagNames・todoSchema の境界値）。自明なコンポーネントのユニットは書かない。

事前準備（Lesson 1 の前に）

  • Node.js 20 以上（LTS 推奨）と pnpm
