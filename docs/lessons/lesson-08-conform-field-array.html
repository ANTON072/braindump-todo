Day 19: field array のフォーム

1. 保存用スキーマ

src/features/braindump/schema.ts に追記。抽出スキーマ（API の都合で nullable）とは別に、フォーム用のスキーマを定義します。タグは行ごとのカンマ区切り文字列にして、field array を「やさしい形」に保ちます。

export const braindumpFormSchema = z.object({
  tasks: z
    .array(
      z.object({
        title: z.string({ required_error: "タイトルは必須です" }).max(200),
        dueDate: z.string().date().optional(),
        priority: z.enum(["low", "med", "high"]).default("med"),
        tagsInput: z.string().max(500).optional(),
      }),
    )
    .min(1, "タスクが 1 件もありません"),
});

2. ページの状態設計

src/app/braindump/page.tsx（Client Component）は 2 段階の画面です：

  1. 入力段階 — textarea＋「抽出」ボタン。runBraindumpExtraction（Lesson 7）を呼ぶ。
  2. 確認段階 — 抽出結果を defaultValue にした field array フォーム。編集して「すべて保存」。

抽出結果はフォームの defaultValue に変換してから渡します（null → undefined、tags[] → カンマ区切り）：

const defaultValue = {
  tasks: extractedTasks.map((task) => ({
    title: task.title,
    dueDate: task.dueDate ?? undefined,
    priority: task.priority ?? "med",

3. field array フォーム本体

src/features/braindump/braindump-form.tsx：

"use client";

import { useActionState } from "react";
import { useForm, getFormProps, getInputProps, getSelectProps } from "@conform-to/react";
import { parseWithZod } from "@conform-to/zod";
import { saveBraindumpTasks } from "./actions";
import { braindumpFormSchema } from "./schema";
import { Button } from "@/components/ui/button";
import { Input } from "@/components/ui/input";

export function BraindumpForm({ defaultValue }: { defaultValue: unknown }) {
  const [lastResult, action] = useActionState(saveBraindumpTasks, undefined);
  const [form, fields] = useForm({
    lastResult,
    defaultValue,
    onValidate({ formData }) {
      return parseWithZod(formData, { schema: braindumpFormSchema });
    },
  });

  const taskFields = fields.tasks.getFieldList();

  return (
    <form {...getFormProps(form)} action={action} className="space-y-4">
      {taskFields.map((taskField, index) => {
        const task = taskField.getFieldset();
        return (
          <div key={taskField.key} className="flex items-start gap-2 rounded border p-3">
            <div className="flex-1 space-y-2">
              <Input {...getInputProps(task.title, { type: "text" })} />
              <p className="text-sm text-red-600">{task.title.errors}</p>
              <div className="flex gap-2">
                <Input {...getInputProps(task.dueDate, { type: "date" })} />
                <select {...getSelectProps(task.priority)}
                        className="rounded border p-2">
                  <option value="low">低</option>
                  <option value="med">中</option>
                  <option value="high">高</option>
                </select>
              </div>
              <Input {...getInputProps(task.tagsInput, { type: "text" })}
                     placeholder="タグ（カンマ区切り）" />
            </div>
            <Button variant="ghost" size="sm"
              {...form.remove.getButtonProps({ name: fields.tasks.name, index })}>
              削除
            </Button>
          </div>
        );
      })}

      <div className="flex gap-2">
        {/* Enter キーの暗黙送信は DOM 上で最初の submit に飛ぶため、保存を先に置く */}
        <Button type="submit">すべて保存</Button>
        <Button variant="outline" type="submit"
          {...form.insert.getButtonProps({ name: fields.tasks.name })}>
          行を追加
        </Button>
      </div>
      <p className="text-sm text-red-600">{fields.tasks.errors}</p>
    </form>
  );
}

学びどころ：Conform の field array を支える 3 つの仕掛け

  • getFieldList() — 配列フィールドを行のリストに展開する。各行の getFieldset() でネストしたフィールド（title 等）に降りる。「配列 → 行 → 各項目」の 2 段降下が基本形。
  • intent ボタン — 「行を追加」「削除」は自前の useState ではなく form.insert / form.remove の getButtonProps で作る。実体は特殊な name を持つ submit ボタンで、フォームの状態変更もフォーム送信の一種として扱うのが Conform の思想。JS が無効でも動くのはこの設計のおかげ。
  • key={taskField.key} — index ではなく Conform が振る key を使う。行削除時に React が行を取り違えるバグ（field array の定番事故）を防ぐ。

Day 20: 一括保存と仕上げ

4. 保存 Server Action

src/features/braindump/actions.ts に追記。Lesson 5 のヘルパーがそのまま再利用できます——ここが総集編たるゆえんです。

import { redirect } from "next/navigation";
import { parseWithZod } from "@conform-to/zod";
import { db } from "@/db";
import { todos } from "@/db/schema";
import { attachTagsToTodo, normalizeTagNames } from "@/features/tags/upsert-tags";
import { braindumpFormSchema } from "./schema";

export async function saveBraindumpTasks(prevState: unknown, formData: FormData) {
  const userId = await requireUserId();

  const submission = parseWithZod(formData, { schema: braindumpFormSchema });
  if (submission.status !== "success") return submission.reply();

  await db.transaction(async (tx) => {
    for (const task of submission.value.tasks) {
      const { tagsInput, ...todoValues } = task;
      const [todo] = await tx.insert(todos)
        .values({ ...todoValues, userId })
        .returning();
      await attachTagsToTodo(tx, userId, todo.id, normalizeTagNames(tagsInput ?? ""));
    }
  });

  redirect("/todos");
}

学びどころ：全件で 1 トランザクション

5 件抽出して 3 件目で失敗したとき、「2 件だけ保存された」状態はユーザーには失敗にしか見えません。 ブレインダンプ 1 回の保存は all-or-nothing にするのが正解で、ループ全体を 1 つの db.transaction で包みます。 Lesson 5 で attachTagsToTodo が tx を受け取る設計にしておいたことが、ここでそのまま効いています。

チェックポイント（＝完成条件）

  • 「歯医者に電話、牛乳買う、金曜までにレポート、Sam にメール」を貼る → 4 行の編集フォームが出る
  • 1 行削除・1 行追加・タグを書き換えてから保存 → /todos に全件がタグ付きで並ぶ
  • 同じタグ名は再利用されている（tags テーブルが重複していない）
  • タイトルを空にした行があると保存できず、その行にエラーが出る
  • 以上すべてが CloudFront URL（本番）でも動く

つまずきポイント

  • 「行を追加」を押したらフォーム全体が送信されてしまう場合、getButtonProps のスプレッドが type="submit" より後に来ているか確認（intent ボタンは特殊な name/value を持つ submit として動く必要がある）。
  • テキスト入力中に Enter を押すと、HTML の暗黙送信の仕様により DOM 上で最初の submit ボタンが押された扱いになる。「行を追加」を保存より先に置くと Enter のたびに行が増えるため、本文のコードでは「すべて保存」を先にしている。
  • 行を削除したら別の行の入力値が消えた／ずれた場合は key に index を使っている。taskField.key に直す。
  • ネストしたフィールドの name は tasks[0].title 形式。parseWithZod はこれを自動で配列に復元する。自前で name を組み立てないこと。
